# SEA Coastal Leaders – Weekly Analysis Pipeline  
This repository hosts the automated weekly analysis of the **94 SEA Coastal Leaders applications**, generated by a Zapier workflow. The purpose of this analysis is not demographic reporting, but to determine whether there is **critical mass, thematic convergence, and network readiness** to justify building a larger **ASEAN grassroots coastal leadership platform**.

The analysis is produced once per week and saved to:

- **analysis-results.md** (auto‑generated)
- **Live page:** *TBD*

---

## Overview  
Every week, Zapier loads the full Excel dataset of 94 applicants and runs a structured analysis to detect **themes, opportunity signals, and structural patterns** across the region. The goal is to generate evidence for:

- Cross-border issue convergence  
- Replicable solution models  
- Policy engagement depth  
- Regional collaboration demand  
- Leadership connectivity and isolation gaps  

These insights help determine whether a regional platform is justified and what shape it should take.

---

## What the Agent Does  
The Zapier Agent performs the following steps:

### 1. Load and Inspect the Dataset  
- Reads the Excel file containing all 94 applications.  
- Identifies all column names.  
- Detects qualitative narrative fields (e.g., problem statement, focus area, impact, challenges, opportunities, collaboration, scaling, motivation).  
- Concatenates relevant narrative fields into a single cleaned text block per applicant (lowercased, punctuation removed, missing values handled).

---

## Metrics Generated Weekly  
The Agent produces **five strategic metrics**, each with:

- A clear definition  
- Numerator, denominator, and percentage  
- A static visual  
- Three anonymized text snippets that triggered categorization  
- Country and subject filters  
- A short narrative explaining how the metric supports the case for a regional platform  

### 1. Cross-Border Issue Convergence Index  
- Creates 8–10 barrier categories using keyword clustering (e.g., overfishing, climate impacts, weak enforcement, market exploitation, pollution, habitat loss, limited finance, exclusion from decision-making).  
- Detects which barriers each applicant mentions.  
- Identifies the top 3 most common barriers.  
- Calculates the percentage of applicants mentioning **at least two** of the top three barriers.  
- **Visual:** Bar chart of top 5 barriers + annotated convergence percentage.

### 2. Replicable Model Density  
- Categorizes solution models (e.g., community-managed marine areas, mangrove restoration, traceability tools, bycatch mitigation, community enterprise, ecotourism, aquaculture improvements, citizen monitoring).  
- Flags model types per applicant.  
- Calculates the percentage implementing **one** replicable model and **two or more**.  
- **Visual:** Histogram of number of model types per applicant.

### 3. Policy Engagement Penetration  
- Detects signals of engagement with government or governance systems (keywords: government, ministry, policy, regulation, council, co-management, ordinance, customary law recognized).  
- Calculates percentage with **at least one** engagement signal and percentage with **multiple** signals.  
- **Visual:** Stacked bar showing none / moderate / strong engagement.

### 4. Peer Learning & Regional Demand Signal  
- Detects language indicating desire for regional collaboration (keywords: ASEAN, regional, cross-border, peer learning, exchange, network, collaboration, platform, partnerships).  
- Calculates percentage expressing explicit interest in cross-country engagement.  
- **Visual:** Bar chart of most frequent collaboration-related terms + headline percentage.

### 5. Leadership Connectivity Gap (Isolation vs Readiness)  
- Detects isolation signals (e.g., underrepresented, lack of funding, limited access, isolated, need visibility, no platform).  
- Calculates percentage of applicants who show **all three**:  
  - Policy engagement  
  - Peer-learning demand  
  - Isolation language  
- Interprets this group as **“ready but structurally disconnected.”**  
- **Visual:** Quadrant or 2×2 classification showing readiness vs connectivity.
